The config looks clean — no `limit_except` — so the method restriction is the unconditional `Connection "upgrade"`. When nginx sends `Connection: upgrade` on every proxied request (including regular POST), backends can reject it with 405 because a POST has no business requesting a protocol upgrade.

The fix is the standard conditional WebSocket pattern: only send `Connection: upgrade` when the client actually requested an upgrade, otherwise send nothing. In NixOS you need a `map` block at the HTTP level plus a change to the location:

```nix
services.nginx = {
  commonHttpConfig = ''
    map $http_upgrade $connection_upgrade {
      default  upgrade;
      ""       "";
    }
  '';

  virtualHosts."dissona.hgeorgiev.com" = {
    forceSSL = true;
    enableACME = true;

    locations."/" = {
      proxyPass = "http://10.100.0.100:7679";
      extraConfig = ''
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Port 443;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;

        client_max_body_size 1000M;

        proxy_set_header Range $http_range;
        proxy_set_header If-Range $http_if_range;
        proxy_force_ranges on;

        proxy_buffering off;
      '';
    };
  };
};
```

The only real change is the `map` block in `commonHttpConfig` and replacing the hardcoded `"upgrade"` with `$connection_upgrade`. WebSocket requests still work (they carry `Upgrade: websocket` so the map resolves to `upgrade`), but plain HTTP requests like `POST /oauth/register` get an empty `Connection` header and are proxied normally.
